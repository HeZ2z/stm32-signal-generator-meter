# AGENTS.md

作用范围：`/home/hz/codes/hw/stm32` 及其所有子目录。

## 任务背景

本目录用于后续开发一项一周周期的 `STM32` 实践项目。

当前确认的项目方向为：
`基于 STM32F429 的信号发生、波形显示与李萨如演示一体机`

如后续用户明确改题，应先更新 `README.md` 与本文件，再继续实现。

## 已确认约束

- 开发板：`Apollo STM32 F4/F7`
- 当前仅有开发板本体
- 无外接传感器
- 项目周期只有 `1 周`
- 题目必须适合开发板独立完成或通过板内回环完成
- 优先保证能演示、能答辩、能在周期内收尾

## 项目目标

在单块开发板上完成如下链路：

`参数设置 -> 信号输出 -> 信号输入 -> 参数测量 -> 结果显示`

当前已完成主线优先实现：

1. `PWM` 方波输出
2. 输入捕获测量
3. 频率、周期、占空比计算
4. 串口输出结果
5. 参数可调

当前后续扩展目标：

1. 真实采样驱动的 `YT` 波形显示
2. 双路 `DAC` 波形输出
3. 双通道 `ADC` 回采
4. `XY` 模式图形显示
5. 可展示李萨如图形

## 设计与实现原则

- 优先做最小可行版本，不要一开始做大而全方案
- 优先依赖板载资源，不引入不必要外设
- 优先保证 `UART` 可用，再推进复杂功能
- 优先验证 `TIM`、`PWM`、`Input Capture` 基本链路
- 在没有跑通板内回环前，不要扩展到复杂显示或次要功能
- 新阶段继续遵循“先跑通真实采样显示，再扩双通道和波形种类”
- 一周周期内，删除复杂度优先于增加功能

## 推荐的最小实现范围

当前第一阶段已交付：

- 一个定时器输出可配置 `PWM`
- 一个输入通道测量该信号
- 串口打印频率、周期、占空比
- 至少一种参数调整方式：按键或串口命令

当前后续阶段按以下顺序推进：

- `M8`：把 LCD 从状态页推进到真实采样 `YT` 波形页，固定使用 `PB6 -> PA0`
- `M9`：切到双路 `DAC` 方波和双通道 `YT`
- `M10`：加入三角波并打通最小 `XY`
- `M11`：加入正弦波并完成李萨如演示

推荐的最小验证方式：

- 使用杜邦线将 `PWM 输出引脚` 连接到 `输入捕获引脚`
- 使用串口助手观察设定值与实测值

## 推荐模块划分

- `signal_gen`：信号产生，负责 PWM 输出参数配置
- `signal_measure`：输入捕获与参数计算
- `signal_gen_dac`：双路 DAC 波形产生
- `signal_capture_adc`：真实采样与 DMA 缓冲
- `scope_render`：`YT / XY` 波形绘制
- `ui_ctrl`：按键或串口命令处理
- `display`：串口输出或板载显示输出
- `app`：主状态机与调度

## 开发优先级

1. 确认准确 MCU 型号与开发板资源
2. 建立最小可编译工程
3. 跑通串口日志输出
4. 跑通 PWM 输出
5. 跑通输入捕获
6. 计算频率、周期、占空比
7. 实现参数调整
8. 做误差分析与边界验证
9. 推进 `M8-M11` 的真实波形显示、双 DAC 与 `XY` 演示

## 禁止事项

- 不要默认存在外接传感器
- 不要在第一版中引入过多外设依赖
- 不要为了“炫技”牺牲可交付性
- 不要在未确认具体芯片型号前硬编码时钟与外设映射
- 不要跳过串口或日志验证直接堆复杂功能

## 验证要求

任何声称“完成”的阶段性工作，至少应给出以下证据之一或组合：

- 成功编译
- 成功烧录
- 串口输出符合预期
- 设定频率与实测频率基本一致
- 设定占空比与实测占空比基本一致
- 对时钟、分频、计数换算有明确说明
- LCD 波形图与真实输入变化一致
- `XY` 模式轨迹与频率比、相位差变化可解释
