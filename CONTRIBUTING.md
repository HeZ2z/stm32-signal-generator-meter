# Contributing

## 适用范围

本仓库当前处于项目起步阶段，目标是先把 `STM32` 单板闭环方案跑通，再逐步扩展功能。

在当前阶段，贡献的优先级高于“功能数量”的是：

- 明确约束
- 降低不确定性
- 保持结构清晰
- 给出可验证的阶段结果

## 提交前请先确认

1. 你是否确认了具体 `MCU` 型号，或者明确标注了“待确认”
2. 你是否避免在未确认芯片前硬编码时钟、外设和引脚映射
3. 你是否优先保证 `UART -> PWM -> Input Capture` 这条最小链路
4. 你是否更新了对应文档和验证记录

## 推荐工作方式

- 先在 `docs/decision-log.md` 记录关键技术选择
- 先在 `docs/pin-plan.md` 填写引脚候选，再落代码
- 每次阶段性完成后更新 `docs/verification-log.md`
- 任何会改变题目、周期、硬件条件的修改，都要同步更新 `README.md` 与 `AGENTS.md`

## 代码与目录约定

- `src/` 放核心实现
- `include/` 放公共头文件
- `tools/` 放辅助脚本
- `cmake/` 放工具链和构建辅助文件
- `build/` 为本地构建产物，不提交到版本库

## 提交建议

- 提交信息尽量聚焦单一主题
- 不要把临时备份、构建输出、串口日志一并提交
- 如果某项功能尚未可用，请在文档中明确说明当前边界

## 验证建议

至少提供以下证据之一：

- 成功编译
- 成功烧录
- 串口输出符合预期
- 设定频率与实测频率基本一致
- 设定占空比与实测占空比基本一致
- 对时钟、分频、计数换算有清晰说明
