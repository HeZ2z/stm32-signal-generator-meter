# Architecture

## 目标链路

当前主固件围绕稳定演示展开：

`参数设置 -> PWM 输出 -> 输入测量 -> UART/LCD 输出`

后续扩展主线为：

`参数设置 -> DAC 输出 -> ADC 采样 -> YT/XY 显示`

## 模块边界

### `app`

- 负责启动顺序、状态管理和周期性调度
- 负责协调各模块之间的调用关系

### `signal_gen`

- 负责 `PWM` 频率和占空比配置
- 对外提供“设定目标波形参数”的统一接口
- 将参数边界检查和定时器换算拆到纯逻辑层，便于宿主机测试

### `signal_gen_dac`

- 负责双路 `DAC` 波形产生
- 当前 `M9` 实现为 `TIM6` 统一触发、单 DMA 向 `DAC->DHR12RD` 写 packed 双通道样本
- 首版固定双方路同频同相 `square 50%`
- `M10` 加入正弦/三角波查表
- `M11` 完成李萨如图形链路

### `ui_ctrl`

- 负责接收触摸输入并把用户交互转成内部配置请求
- 当前 `UART` 退化为只读状态口，不再直接改写参数
- 保持一份 LCD 控制页视图状态，供 `display_lcd` 增量刷新
- 当前 `MORE` 作为 LCD 项目信息弹层开关，不再复用 UART help 语义

### `signal_measure`

- 负责 `TIM PWM Input` 初始化和捕获中断处理
- 负责输出最近一次有效测量结果
- 将计数值换算拆到纯逻辑层，便于宿主机测试

### `signal_capture_adc`

- 负责真实波形采样、DMA 环形缓冲和快照抽取
- `M8` 先从 `PB6 -> PA0` 单通道真实采样起步
- `M9` 起扩展到 `ADC1` 双 rank 扫描、交错 DMA 和双通道 frame 输出
- 当前触发源实际为 `TIM2 TRGO`，因为 `STM32F429` 常规 ADC 触发列表不含 `TIM7`

### `scope_render`

- 负责把采样缓冲映射到 LCD 波形视图
- 提供 `YT` 和 `XY` 两种绘制路径
- `M10` 先接入最小 `XY`，`M11` 完成李萨如演示视图

### `display`

- 负责统一显示输出接口
- 当前 `UART` 与 `ALIENTEK 4342 RGBLCD` 并行工作，LCD 负责板载演示，UART 保留兜底调参
- LCD 后端当前负责开机图、状态页、触摸按键页和 `MORE` 项目信息弹层
- 后续会把状态页扩展为真实波形页，并增加 `YT / XY` 绘图区域

### `touch`

- 负责 `GT9xxx` 电容触摸初始化、轮询和坐标读取
- 负责把原始坐标映射为 LCD 坐标并输出触摸事件
- 当前增加了短窗口按下/释放去抖，抑制边缘抖动导致的连点
- 当前不做电阻触摸校准与 Flash 持久化

## 推荐数据流

1. `touch` 采样并输出触摸事件
2. `ui_ctrl` 接收参数修改请求
3. `app` 调用 `signal_gen` 更新输出参数
4. `signal_gen` 输出新的 `PWM`
5. `signal_measure` 对回环信号做捕获与换算
6. `display` 刷新 LCD 控制页并通过串口输出同步状态

后续扩展数据流：

1. `signal_gen_dac` 通过 `TIM6 + DMA` 输出 `PA4/PA5`
2. `signal_capture_adc` 通过 `TIM2 TRGO + ADC1 scan + DMA` 回采 `PA0/PA6`
3. `scope_render` 输出双通道 `YT`
4. 后续 `M10/M11` 在此基础上扩展 `XY`

## 当前实现边界

- 当前按 `STM32F429` 路线收敛，时钟默认使用 `HSI 16 MHz`
- 当前已接入真实 `LTDC + SDRAM + RGB565` 最小 LCD 后端
- 当前 `PB5` 已从 `TIM3_CH2` 测量输入迁出，固定改作 `LCD_BL`
- 当前不做复杂波形合成，首版只做 `PWM` 方波
- 当前不引入外接传感器与外部信号源
- 当前测试以宿主机侧纯逻辑回归为主，不做板级自动化测试
- 当前 `M8` 已完成单通道真实采样 `YT`，`display_lcd.c` 已按技术层拆分为 `lcd_prim.c`（绘图原语）和 `lcd_font.c`（字模）
- 双通道模拟链路固定按 `PA4/PA5` 输出、`PA0/PA6` 回采规划
- 双通道采样首版接受顺序扫描的非严格同时采样限制，主演示频率优先控制在中低频
- 旧 `PWM + Input Capture` 代码仍保留在构建中，但默认不初始化，只作为 dormant 回归链路

## 关键风险

- 若实板 `PB5/TIM3_CH2` 不可用，需要调整输入捕获映射
- 当前 `MEAS` 的有效性依赖回环线与中断工作正常
- 当前没有板级自动化测试，最终闭环结果仍需人工联调确认
- 当前 LCD 仍为单层 `RGB565` + CPU 绘制，不做 `DMA2D` 和双缓冲
- 若 LCD 初始化失败，系统退化为 `UART only`，不影响 PWM、测量和串口命令主线
- `XY` 轨迹显示质量会受到采样同步方式和演示频率范围影响
