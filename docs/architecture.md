# Architecture

## 目标链路

当前主固件围绕稳定演示展开：

`参数设置 -> PWM 输出 -> 输入测量 -> UART 输出`

## 模块边界

### `app`

- 负责启动顺序、状态管理和周期性调度
- 负责协调各模块之间的调用关系

### `signal_gen`

- 负责 `PWM` 频率和占空比配置
- 对外提供“设定目标波形参数”的统一接口
- 将参数边界检查和定时器换算拆到纯逻辑层，便于宿主机测试

### `ui_ctrl`

- 负责接收串口命令或按键输入
- 负责把用户输入转成内部配置请求
- 命令解析与缓冲规则拆到独立纯逻辑层，避免和串口硬件耦合

### `signal_measure`

- 负责 `TIM PWM Input` 初始化和捕获中断处理
- 负责输出最近一次有效测量结果
- 将计数值换算拆到纯逻辑层，便于宿主机测试

### `display`

- 负责当前显示输出实现，现阶段为 `UART`
- 后续可在同一模块边界下扩展为板载屏幕输出

## 推荐数据流

1. `ui_ctrl` 接收参数修改请求
2. `app` 调用 `signal_gen` 更新输出参数
3. `signal_gen` 输出新的 `PWM`
4. `signal_measure` 对回环信号做捕获与换算
5. `display` 通过串口输出设定值与实测值

## 当前实现边界

- 当前按 `STM32F429` 路线收敛，时钟默认使用 `HSI 16 MHz`
- 当前不引入 `OLED/LCD` 等额外显示模块
- 当前不做复杂波形合成，首版只做 `PWM` 方波
- 当前不引入外接传感器与外部信号源
- 当前测试以宿主机侧纯逻辑回归为主，不做板级自动化测试

## 关键风险

- 若实板 `PB5/TIM3_CH2` 不可用，需要调整输入捕获映射
- 当前 `MEAS` 的有效性依赖回环线与中断工作正常
- 当前没有板级自动化测试，最终闭环结果仍需人工联调确认
