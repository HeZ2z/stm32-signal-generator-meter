# Architecture

## 目标链路

当前主固件围绕稳定演示展开：

`参数设置 -> PWM 输出 -> UART 输出`

## 模块边界

### `app`

- 负责启动顺序、状态管理和周期性调度
- 负责协调各模块之间的调用关系

### `signal_gen`

- 负责 `PWM` 频率和占空比配置
- 对外提供“设定目标波形参数”的统一接口

### `ui_ctrl`

- 负责接收串口命令或按键输入
- 负责把用户输入转成内部配置请求

### `display`

- 负责通过 `UART` 输出当前设定值和状态信息
- 后续如果资源允许，可扩展为板载屏幕输出

## 推荐数据流

1. `ui_ctrl` 接收参数修改请求
2. `app` 调用 `signal_gen` 更新输出参数
3. `signal_gen` 输出新的 `PWM`
4. `display` 通过串口输出结果

## 当前实现边界

- 尚未确认具体 `MCU` 型号，因此不写死时钟树与外设实例
- 当前不引入 `OLED/LCD` 等额外显示模块
- 当前不做复杂波形合成，首版只做 `PWM` 方波
- 当前不引入外接传感器与外部信号源
- 当前不在主固件中保留未稳定的测量实现

## 关键风险

- 板卡具体 `MCU` 型号未确认，可能影响 `TIM` 和引脚映射选择
- 若优先选择 `F7`，需要额外注意时钟、缓存和工程复杂度
- 若后续重新引入测量链路，应优先以独立实验入口验证，而不是直接叠加到主固件
