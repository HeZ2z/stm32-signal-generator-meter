# Decision Log

## 2026-04-14

### 决策 1

- 主题优先选择：`基于 STM32F4/F7 的信号发生与频率测量一体机`
- 原因：不依赖外接传感器，单板即可构成完整闭环，适合一周周期

### 决策 2

- 首版显示路径优先使用：`UART`
- 原因：开发成本低，调试效率高，适合快速验证测量链路

### 决策 3

- 在确认准确 `MCU` 型号前，不提交与具体 `F4/F7` 型号强耦合的初始化代码
- 原因：避免错误的时钟树、外设实例和引脚映射带来返工

### 决策 4

- 首版最小演示方式采用：`PWM 输出 + 输入捕获 + 板内回环`
- 原因：验证路径最短，风险最低，结果最容易解释

### 决策 5

- 首个可实现工程临时按：`STM32F429 + HAL + CMake`
- 原因：Apollo `F4/F7` 实物型号尚未确认，但需要先把工程、构建链和模块边界落地

### 决策 6

- 首版系统时钟优先使用：`HSI -> PLL -> 180 MHz`
- 原因：避免在外部晶振频率未确认前，把工程启动依赖绑死到 `HSE`

### 决策 7

- 主固件当前收口为：`UART + PWM + 串口命令 + LED`
- 原因：该链路已验证稳定，适合作为阶段性可演示版本

### 决策 8

- 未稳定的测量实现从主线移除
- 原因：避免影响主固件稳定性，后续测量功能单独实验
