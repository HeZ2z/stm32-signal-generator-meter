# Roadmap

## 当前状态

- 当前主线阶段：`M5 完成`
- 当前扩展阶段：`M6 准备中`

说明：

- 主固件已经集成 `UART + PWM + TIM PWM Input + 串口命令 + LED`
- `PWM` 输出、串口参数调整和测量结果换算已经可构建、可测试
- `PB6 -> PA7` 实板回环已经验证通过，默认测量和断线退化行为符合预期
- `M5` 的误差展示、演示脚本和输出格式收尾已完成
- 当前优先事项切换为 `ALIENTEK 4342 RGBLCD` 集成，Apollo F429 屏参和 `SDRAM Bank1` 基线已确认，待继续实板点屏验证

## M0 仓库与文档初始化

目标：

- 完成目录规范
- 明确项目边界
- 建立设计、计划和验证文档

退出条件：

- `README.md` 可直接说明项目在做什么
- `docs/` 中具备路线图、设计、引脚和验证模板

## M1 板卡确认与最小工程建立

目标：

- 确认准确 `MCU` 型号
- 确认调试器、串口和基础时钟方案
- 生成最小工程

退出条件：

- `docs/pin-plan.md` 填入关键引脚
- 能完成一次最小工程编译

## M2 UART 跑通

目标：

- 通过串口输出启动日志和基础状态

退出条件：

- 串口助手可稳定看到启动日志
- 波特率、时钟来源和端口选择有记录

## M3 PWM 跑通

目标：

- 输出可配置 `PWM`

退出条件：

- 改变配置后波形有可观察变化
- 频率和占空比换算方式有说明

## M4 输入捕获与参数计算

目标：

- 测量频率、周期和占空比

退出条件：

- 板内回环稳定工作
- 串口可输出设定值与实测值

## M5 参数调整与联调

目标：

- 增加串口命令或按键配置能力
- 优化输出格式和错误处理

退出条件：

- 至少支持一种参数修改方式
- 能展示参数修改前后的实测变化

## M6 ALIENTEK 4342 RGBLCD 集成

目标：

- 保留 `UART` 调试链路
- 为 `ALIENTEK 4342 RGBLCD` 接入初始化与状态显示
- 在屏幕上展示 `SET / MEAS / ERR / no-signal`

退出条件：

- 屏幕成功点亮并显示基础状态
- 串口和屏幕输出同时可用
- `docs/verification-log.md` 中有一次上屏验证记录

## 当前建议顺序

1. 继续按 Apollo F429 真值验证 `SDRAM Bank1 + 0xC0000000 + 4342 1/1/40/8/5/8` 路线
2. 在 `display/` 模块内完成点屏和状态页联调，保持 `UART` 不退化
3. 记录一次成功上屏验证或继续收敛剩余 LTDC 问题
