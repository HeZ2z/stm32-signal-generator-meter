# Roadmap

## M0 仓库与文档初始化

目标：

- 完成目录规范
- 明确项目边界
- 建立设计、计划和验证文档

退出条件：

- `README.md` 可直接说明项目在做什么
- `docs/` 中具备路线图、设计、引脚和验证模板

## M1 板卡确认与最小工程建立

目标：

- 确认准确 `MCU` 型号
- 确认调试器、串口和基础时钟方案
- 生成最小工程

退出条件：

- `docs/pin-plan.md` 填入关键引脚
- 能完成一次最小工程编译

## M2 UART 跑通

目标：

- 通过串口输出启动日志和基础状态

退出条件：

- 串口助手可稳定看到启动日志
- 波特率、时钟来源和端口选择有记录

## M3 PWM 跑通

目标：

- 输出可配置 `PWM`

退出条件：

- 改变配置后波形有可观察变化
- 频率和占空比换算方式有说明

## M4 输入捕获与参数计算

目标：

- 测量频率、周期和占空比

退出条件：

- 板内回环稳定工作
- 串口可输出设定值与实测值

## M5 参数调整与联调

目标：

- 增加串口命令或按键配置能力
- 优化输出格式和错误处理

退出条件：

- 至少支持一种参数修改方式
- 能展示参数修改前后的实测变化

## M6 误差分析与演示收尾

目标：

- 做误差与边界测试
- 整理演示流程与答辩材料

退出条件：

- 完成一次完整演示
- `docs/verification-log.md` 中有关键结果记录
